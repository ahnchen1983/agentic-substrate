# Clearing - 清除

## 一句話

Clearing 是主動清理混亂 context、過期假設或錯誤方向，讓工作重新回到清楚狀態的動作。

## 不要誤會成

它不是刪掉所有記憶，也不是逃避釐清問題。

## 為什麼重要

它是長 session 中恢復品質的重要手段，特別是在多次修改、需求轉向或輸出開始偏題時。

## 在五層模型的位置

主要對應 L4 Agent 協調層、L5 Interface 使用介面層。理解它可以幫助你判斷問題發生在五層模型的哪個位置。

## 課程中會出現在哪裡

這個術語會出現在 Day 3 把自己的工作方法交給 AI 協作，用來幫助學員把實作經驗連回 AI 工作系統的概念。

## Agent 需要注意什麼

清理前要保留有效決策；清理後要重新列出目標、目前狀態、限制和下一步。

## 情境例句

> 「這個 Agent 現在一直卡在錯方向，怎麼辦？」
>
> 「可以 clearing：清掉過期假設，重新提供目標、限制和目前狀態。」

## 常見錯誤

- 只說「重來」卻沒有保留有效決策。
- 清掉 context 後忘了補回必要資料。
- 把所有問題都用 clearing 解，沒有修正根本流程。
