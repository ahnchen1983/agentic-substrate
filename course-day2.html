Website Workflow

一頁式網站工具鏈：從想法到可展示頁面

個人簡介 -> 一頁式網站 -> 社群貼文

把履歷、作品、專長與想被誰看見整理成網站敘事，再轉成可放進建站工具的區塊文案。最後由本人確認語氣與事實。

ChatGPT / Claude 整理定位 -> Perplexity 補市場語言 -> Canva / Google Sites / Framer 建頁 -> 人類確認 -> 產出社群貼文
Reverse Challenge

反向挑戰：先不用 AI，你會怎麼做出網站？

Step 1

不用 AI，你會怎麼做？蒐集素材、寫文案、找參考、排版、做圖、上架，全部列出來。

Step 2

哪些環節最花時間？通常是定位不清、文案卡住、版面不知道怎麼排、社群貼文不會寫。

Step 3

哪些交給 AI？哪些仍由人判斷？AI 可起草與組織，人要確認定位、事實、承諾與品牌感。

今天不是做漂亮模板，而是做一個「可以對外說清楚你是誰、提供什麼、下一步怎麼聯絡」的最小可展示頁。
Website Tracks

四種網站題目：選一種當天做完

A. 個人介紹頁

適合學生、求職者、顧問、業師：讓別人快速理解你能幫什麼忙。

B. 公司 / 工作室頁

適合品牌、團隊、服務提供者：說清楚服務、對象、案例與聯絡方式。

C. 商品 / 服務頁

適合產品、課程、工作坊：說清楚痛點、價值、流程、價格或報名方式。

D. 流程 / 提案頁

適合青年提案、內部流程、活動專案：把執行計畫變成外部可讀的一頁說明。

Trap Warning

哪些任務不適合直接交給 AI？

未脫敏資料

客戶個資、公司機密、真實合約資料不要直接貼。

無法驗證的輸出

你不知道對錯，就不該直接採用。

高風險決策

法律、醫療、財務不能完全交給 AI。

成本倒掛

Prompt 設定比人工執行還久，就不值得。

敏感關係

需要信任、同理、承諾的場景，人要站前面。

判斷原則

AI 可以輔助，但人類負責決策與關係。

Closing

Day 3 預告：怎麼讓這次網站流程下次也能用？

今天你做出一個一頁式網站內容包。明天要把這條工具鏈文件化、標準化、可複用化，變成一張 AI 能真的執行的工作流卡。

核心句：工具會變，決策邏輯不變；成果要能帶走，流程要能複用。
