# Day 2 最終版 - AI 工具的選擇與搭配

Status: Working final

核心目的：

> 讓學員學會依任務結構選工具組合，並在當天完成一個可展示的一頁式網站內容包。

## Final Run of Show

| 時間 | 環節 | 重點 |
|---|---|---|
| 0-5 | 安全紅線 + Day 1 回顧 | 公司機密、個資、重要決策不可直接交給公開 AI |
| 5-20 | AI Tool Picker | 探索/生產、是否需要最新資料、是否需要長流程 |
| 20-35 | 反向挑戰 | 不用 AI 做網站會怎麼做？AI 幫在哪？哪裡仍需人類判斷？ |
| 35-50 | 一頁式網站工具鏈示範 | 輸入 -> 工具串接 -> 輸出 -> 人工覆核點 |
| 50-95 | 實作工作坊 | 產出自己的網站內容包、建站提示詞、社群貼文 |
| 95-110 | 同儕檢查 | 清楚嗎？可信嗎？手機可讀嗎？CTA 明確嗎？ |
| 110-120 | Day 3 銜接 | 把今天的網站工具鏈變成可複用 Skill |

## Website Tracks

學員擇一完成：

- 個人介紹頁：學生、求職者、顧問、業師
- 公司 / 工作室頁：品牌、團隊、服務提供者
- 商品 / 服務頁：商品、課程、工作坊、活動
- 流程 / 提案頁：青年提案、內部流程、地方創生專案

## One-Page Website Content Package

每位學員至少完成：

- 網站對象
- 目標受眾
- 核心價值
- 可信證據或案例
- 頁面區塊：首屏、問題、解法、證據、流程、FAQ、CTA
- 建站工具提示詞
- 社群貼文草稿
- 人工覆核點

## Instructor Notes

- Day 2 不再只是案例庫或工具介紹，而是成果導向工作坊。
- 「客戶回饋 -> 問題聚類 -> 改善方案」可轉成改善方案頁，不必放棄原案例。
- 青年提案練習可轉成「流程 / 提案頁」，與一頁式網站主線整合。
- Day 3 銜接語：今天你做出成果，明天把這條流程變成可重複使用的 Skill。
