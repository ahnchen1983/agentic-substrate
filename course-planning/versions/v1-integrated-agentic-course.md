# v1 - 整合版：從聊天到協作，打造你的 AI 工作系統

## 版本來源

這一版整合了幾個來源：

- Agentic Substrate 的學習路徑
- 學生回饋中的互動式課堂節奏
- 跨堂安全、時間配置與學員產出的提醒
- Day 1 改成更普世的震撼 Demo 任務
- Day 2 加入 AI Tool Picker 工具決策框架
- Day 3 加入工作流卡、交換實測與現場跑卡片

## 課程總覽

| 項目 | 說明 |
|---|---|
| 課程名稱 | 從聊天到協作：打造你的 AI 工作系統 |
| 副標 | 基於 Agentic Substrate 架構，從工具使用者升級為系統設計者 |
| 課程規格 | 3 堂 x 2 小時，10:00-12:00 |
| 建議人數 | 小班制，建議 20 人內，方便實作回饋 |
| 目標受眾 | 已有基礎辦公/專案經驗，但未系統性使用 AI 的職場工作者、青年創業者、地方創生從業人員 |
| 核心記憶點 | 你不是在用 AI，你是在設計一個會幫你工作的系統。 |

## 核心敘事

| 日期 | 核心訊息 | 學員轉換 |
|---|---|---|
| Day 1 | AI 是系統 | 從聊天工具使用者，變成工作系統觀察者 |
| Day 2 | 你在設計系統 | 從工具選擇者，變成工作流設計者 |
| Day 3 | 你成為系統架構師 | 從 AI 使用者，變成 AI 協作設計者 |

### 首頁基礎定義與承諾

- 核心承諾：從「AI 是聊天工具」升級到「AI 是新型工作系統」。
- 收尾承諾：三天結束時，你會從工具使用者，變成 AI 協作的設計者。

## 跨堂執行規則

- Day 1 開場 3 分鐘先講三條安全紅線。
- Day 2、Day 3 開場簡短複誦安全提醒。
- 課堂節奏盡量維持：20% 觀念、50% 示範、30% 實作與回饋。
- 課前先寄講義、帳號檢查與網路檢查清單。
- 每堂課都要讓學員帶走一個可以立即使用的產出。
- 課後可建立 Notion / Airtable 模板庫，讓學員上傳與迭代自己的工作流卡。

## 三條安全紅線

Day 1 開場 3 分鐘使用，Day 2 和 Day 3 開場簡短複誦。

```text
1. 公司機密資料、客戶個資、敏感資訊，不要貼進公開 AI 工具。
2. AI 的輸出永遠要有人類覆核。
3. 法律、醫療、財務等高風險決策，不能完全交給 AI。
```

建議講師話術：

> 在我們開始之前，先記住三條紅線。只要你記住這三條，今天回去就可以開始用 AI，而且比較不會出事。第一，公司機密和客戶個資不要直接貼進公開 AI。第二，AI 的輸出永遠要人類覆核。第三，法律、醫療、財務這類高風險決策，不能完全交給 AI。

## Day 1：重新認識生成式 AI，看懂 AI 的新位置

日期：115/05/14（四）10:00-12:00

核心目的：

> 讓學員從「AI 是聊天工具」升級到「AI 是新的工作系統入口」。

### Day 1 時間分配

| 時間 | 環節 | 重點內容 | 教學形式 |
|---|---|---|---|
| 0-3 | 安全紅線 | 機密資料、人類覆核、高風險決策 | 講師開場提醒 |
| 3-10 | 開場與核心記憶點 | 課程承諾：不是只用 AI，而是設計一個會幫你工作的系統 | 快速投票 / 互動提問 |
| 10-30 | 認知升級 | 時代演進框架 + 從搜尋、聊天、產出，到推理與執行 | 視覺化說明 + 案例 |
| 30-50 | 三層 / 五層模型 | `LLM -> Tools -> Skill -> Agent -> Interface`；Skill 是人類經驗的封裝 | 圖層說明 + 口訣記憶 |
| 50-80 | 震撼 Demo | 同一任務，三種做法：傳統、單次聊天、工作流 / Agentic 做法 | 即時示範 |
| 80-95 | 流程成本觀察 | 比較步驟數、誰建結構、能否複用、能否檢查 | 學員觀察表 |
| 95-110 | 即時體驗 | 學員用 2-3 種 AI 做法完成一個小任務 | 小組實作 |
| 110-120 | 邊界收束與 Day 2 預告 | AI 在任務、情境、標準、覆核清楚時最強；預告工具組合 | Q&A + 收尾 |

### Day 1 認知升級：生成式 AI 到底改變了什麼

時代演進框架：

| 時代 | 基本單元 | 代表 | 人的角色 |
|---|---|---|---|
| 桌面時代 | Application | Word、Excel | 操作者 |
| 行動時代 | App | Uber、Line | 使用者 |
| 雲端時代 | Service | AWS、Google Workspace | 設定者 |
| 代理時代 | Skill | ChatGPT、Cursor、Agent | 設計者 |

從搜尋到執行的四個階段：

1. 搜尋：Google 給你十個連結，你自己讀。
2. 聊天：ChatGPT 給你一個答案，你問它答。
3. 產出：AI 直接幫你寫完一篇文章、做完一份簡報。
4. 推理與執行：AI 自己拆任務、自己查資料、自己跑完一整串流程。

關鍵訊息：

> 你現在大概在第 2 階段，三天後你會到第 4 階段。

五層白話比喻：

| 層 | 白話 | 比喻 |
|---|---|---|
| LLM | 理解你說什麼、做推理 | 大腦 |
| Tools | 查網頁、讀檔案、畫圖、寄信 | 四肢 |
| Skill | 你教它「照我的方式做這件事」 | 肌肉記憶 |
| Agent | 自己決定用哪些 Skill、自己跑完 | 經理人 |
| Interface | 聊天框、Cursor、Notion 等你看到的入口 | 辦公桌 |

講師口訣：

> 大部分人在用的只是大腦跟辦公桌。三天後你會開始訓練它的肌肉記憶。

關鍵區分：

> 傳統 App 是確定性的：你按儲存，它就儲存。AI 是概率性的：同樣的問題問兩次，可能給你不同的答案。這不是 bug，這是它的本質。所以你需要學會設計檢查機制，而不是期待它每次都對。

### Day 1 震撼 Demo

不要用太特定的「雲林小農 3 天 2 夜」當第一堂主 Demo，改用更普世的任務：

> 下週要辦一場 30 人的活動，請幫我規劃活動流程、預算、人力分工、邀請文案與報名表單。

為什麼這個任務更適合 Day 1：

- 學生：社團活動、系上活動、成果發表
- 上班族：部門會議、教育訓練、客戶活動
- 創業者：產品發表、招商說明會、小型交流會
- 地方創生：社區活動、工作坊、導覽活動

三種做法：

1. 傳統流程：搜尋範例、開 Word/Excel、手動拆流程、估預算、寫文案、設計表單欄位
2. 單一 ChatGPT 對話：一次 prompt 請 AI 產出活動規劃與文案
3. 工作流 / Agentic 做法：拆成活動目標、流程設計、預算估算、人力分工、邀請文案、報名表欄位、檢查清單與覆核點

比較重點：

- 花幾步？
- 誰負責建立結構？
- 結果能不能複用？
- 結果能不能被檢查？
- 下次辦類似活動能不能直接改版？

### Day 1 學員產出

- 一張簡化的 AI 地圖：`LLM -> Tools / Agents -> Workflow`
- 一份流程成本觀察表
- 對 AI 邊界的合理預期
- 對 Day 2 的問題意識：工具這麼多，到底怎麼選？
- 回家作業：把一個工作上的重複性任務丟給常用 AI，記錄它哪裡做得好、哪裡做得爛。

## Day 2：AI 工具的選擇與搭配，選對使用方式

日期：115/05/15（五）10:00-12:00

核心目的：

> 讓學員不再問「哪個 AI 最強」，而是學會依任務結構選工具組合，並在當天完成一個可展示的一頁式網站內容包。

### Day 2 時間分配

| 時間 | 環節 | 重點內容 | 教學形式 |
|---|---|---|---|
| 0-5 | 安全提醒 + Day 1 複習 | 三條紅線 + AI 是系統 | 快速回顧 |
| 5-20 | AI Tool Picker | 三問框架：探索/生產、是否需要最新資訊、是否需要長流程 | 決策樹 + 範例 |
| 20-35 | 反向挑戰 | 不用 AI 做一頁式網站會怎麼做？AI 幫在哪？哪裡仍需人類判斷？ | 小組討論 + 任務拆解 |
| 35-50 | 一頁式網站工具鏈示範 | 個人/公司/工作室/商品/流程如何從輸入、工具串接到輸出 | 現場示範 |
| 50-95 | 實作工作坊 | 完成自己的網站定位、區塊文案、建站提示詞、社群貼文與覆核點 | 個人實作 + 講師巡場 |
| 95-110 | 同儕檢查 | 清楚嗎？可信嗎？手機可讀嗎？CTA 明確嗎？ | 兩兩回饋 |
| 110-120 | 收尾與 Day 3 預告 | 今天的網站流程，明天變成可複用 Skill | 金句收束 |

### AI Tool Picker：三問選工具

1. 這件事是「探索」還是「生產」？
2. 是否需要最新資訊？
3. 是否需要長流程、多步驟？

| 需求 | 工具方向 |
|---|---|
| 探索 + 最新資訊 | Perplexity / Gemini |
| 深度寫作 / 結構整理 | Claude / ChatGPT |
| 長流程 / 自動化 | Agent / Zapier / n8n |
| 視覺輸出 | Canva / 生成設計工具 |

### 工具組合案例庫

案例庫改為支援主線產出：學員當天完成自己的一頁式網站。案例不是單獨看懂，而是作為實作參考：

1. 研究 -> 摘要 -> 決策備忘錄
2. 會議紀錄 -> 行動項目 -> 跟進信件
3. 地方活動發想 -> 一頁式活動頁 -> 報名 CTA
4. 個人簡介 -> 一頁式網站 -> 社群貼文
5. 客戶回饋 -> 問題聚類 -> 改善方案頁

網站題目可開放四種軌道：

1. 個人介紹頁：學生、求職者、顧問、業師
2. 公司 / 工作室頁：品牌、團隊、服務提供者
3. 商品 / 服務頁：商品、課程、工作坊、活動
4. 流程 / 提案頁：青年提案、內部流程、地方創生專案

雲林小農案例可以放在這裡，作為地方創生或活動頁案例：

> 地方活動發想 -> 行程設計 -> 一頁式活動頁 -> 宣傳文案 -> 報名 CTA

### 一頁式網站內容包

每位學員至少完成：

- 網站對象：個人 / 公司 / 工作室 / 商品 / 服務 / 流程 / 提案
- 目標受眾：誰會看，對方現在卡在哪裡
- 核心價值：我能幫他前進哪一步
- 區塊結構：首屏、問題、解法、證據、流程、FAQ、CTA
- 建站提示詞：可貼進 Canva、Google Sites、Framer、Carrd、Gamma 或 AI 建站工具
- 社群貼文：用來發布這個頁面
- 人工覆核點：事實、承諾、機密、個資、手機可讀性、CTA

### 陷阱警示

這一段不是第一次講安全，而是放在「工具選擇風險」裡加深辨識：

- 未脫敏的機密資料
- 學員無法驗證正確性的輸出
- 法律、醫療、財務等高風險任務
- Prompt 設定成本高於人工執行成本
- 需要人類信任、同意或敏感關係處理的任務

## Day 3：AI 生成靈魂融入日常，把自己的方法交給協作

核心目的：

### Day 3 時間分配

| 時間 | 環節 | 重點內容 | 教學形式 |
|---|---|---|---|

| 10-30 | 什麼是 Skill | Skill 定義、解剖學、Prompt vs Skill | 表格講解 + 會議紀錄範例 |
| 30-50 | 七問萃取法 | 回憶練習 + 七問框架 | 個人書寫 |
| 50-90 | 完成 AI 工作流卡 | 學員現場填寫，講師巡場 | 模板引導 + 一對一諮詢 |
| 90-110 | 交換實測 | 學員交換卡片，嘗試跑對方流程 | 兩兩配對 + 回饋表 |

| 140-160 | 版本迭代 + 90 天計畫 | v1 卡片 -> Prompt -> 檢查表 -> 半自動流程 -> Skill / Agent | 路徑圖 + 個人規劃 |
| 160-180 | 結業儀式 | 核心金句 + 社群銜接 | 收束 |

### 什麼是 Skill

Skill 定義：

> Skill = 你的經驗 + 結構化的流程 + AI 可以執行的指令。

Skill 解剖學：

| 組成 | 白話 | 範例：整理會議紀錄 |
|---|---|---|
| 觸發條件 | 什麼時候啟動 | 收到會議錄音或逐字稿時 |
| 需要什麼輸入 | 你要給 AI 什麼東西 | 逐字稿、與會者名單、會議主題 |
| 步驟 | 一步一步怎麼做 | 擷取重點 -> 分類決議/待辦 -> 指派負責人 -> 格式化 |
| 產出長什麼樣 | 最終成果格式 | 待辦事項 / 負責人 / 截止日表格 |
| 怎麼檢查好壞 | 驗收標準 | 每個待辦都有明確負責人和日期 |
| 前後接什麼 | 上游跟下游 | 前：會議結束；後：寄信、更新專案看板 |

Prompt vs Skill：

|  | Prompt | Skill |
|---|---|---|
| 時間 | 一次性 | 可重複使用 |
| 範圍 | 一個問題 | 一個完整流程 |
| 誰的知識 | AI 的通用知識 | 你的專業知識 |
| 品質控制 | 通常沒有 | 有檢查標準 |

### 反向設計框架

不要只問：

> 我要怎麼用 AI 完成這件事？

更要問：

> AI 要做好這件事，需要知道什麼？

建議講師話術：

> 當你要把一件事交給 AI 協作，你先要能清楚回答七個問題。這七個問題不是給 AI 看的，而是訓練你自己把隱性經驗說清楚。說不清楚，就代表你還沒真的掌握這件事。

### 七問 AI 工作流卡

1. 這件事是什麼？任務定義是什麼？
2. 什麼時候開始？觸發條件是什麼？
3. 需要哪些輸入？格式或範例是什麼？
4. 步驟是什麼？執行流程怎麼走？
5. 產出長什麼樣？輸出標準是什麼？
6. 怎麼檢查好壞？人工覆核點在哪裡？
7. 前後接什麼流程？串接點是什麼？

### 交換實測

學員兩兩配對：

1. 交換工作流卡。
2. 嘗試用對方的卡片跑一次流程。
3. 回饋：
   - 少了哪些必要輸入？
   - 哪些步驟不夠清楚？
   - 驗證規則是否足夠？

教學重點：

> 好的工作流，是別人也能操作的。

### 現場產出儀式

1. 學員把工作流卡貼到共用 Notion / Airtable / Google Doc。
2. 講師挑 3 張卡片，盡量涵蓋不同族群：學生、上班族、創業者。
3. 講師現場用真實 AI 工具跑一次，把產出投影給全班看。
4. 邀請卡片作者分享：這個產出跟你預期的落差在哪？哪裡可以優化？

關鍵體感：

> 我寫的這張卡，真的可以變成 AI 的工作。

### 工作流版本迭代路徑

```text
工作流卡 v1 -> 優化 Prompt -> 可重複檢查表 -> 半自動流程 -> Skill / Agent
```

## 講師設計備忘

- 優先使用有在地感的案例：小農品牌、社區導覽、青年提案、地方產品故事。
- 每堂課都要讓學員帶走具體產物。
- 核心記憶點要用不同形式重複。
- 先講安全，再講熱情。
- 強調迭代：工作流是透過使用逐步變好的。

## 最終目標

學員離開時，帶走的不只是知識，而是一套可以持續進化的個人協作系統。
